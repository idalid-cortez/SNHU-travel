## Reflexión sobre el programa de CS

**¿Cómo debo interpretar las necesidades del usuario y las implemento en un programa? ¿Cómo ayuda esto a la creación de “historias de usuario”?**
Interpreto las necesidades del usuario analizando los problemas reales del cliente y traduciéndolos a requisitos técnicos. Este proceso es fundamental para crear "historias de usuario", ya que permite segmentar las funcionalidades en tareas pequeñas, concretas y centradas en el valor que recibe el usuario final.

**¿Cómo debo abordar el desarrollo de programas? ¿Qué procesos agile espero incorporar en mi futuro trabajo de desarrollo?**
Abordo el desarrollo mediante un enfoque iterativo, donde la entrega incremental permite ajustar el rumbo ante cambios. En mi futuro profesional, espero incorporar procesos Scrum, como los "sprints" para organizar el tiempo, "daily meetings" para la transparencia y retrospectivas para garantizar la mejora continua de mis flujos de trabajo.

**¿Qué significa ser un buen miembro del equipo de desarrollo de software?**
Ser un buen miembro implica fomentar una comunicación clara y proactiva, practicar la responsabilidad compartida y participar constructivamente en las revisiones de código. Un desarrollador efectivo no solo escribe código funcional, sino que documenta su trabajo y prioriza el éxito colectivo sobre el individual.
